AIDE DroidScript Plugin Generator

This is a quick guide to get you started in DS plugin development using the AIDE (Android Java IDE) app. This is not a tutorial about java or AIDE. You must have AIDE installed before this script will work.


Creating a New Plugin Template

Select Create New DS Plugin from the menu. You will be prompted to enter a plugin name and the name you want to use for the individual/company package name identifier.

Plugin name should be short, using the camel naming convention. As example: 'MyPlugin'.

Individual/Company name will represent the package name. This should be one word. Example: 'mycompany'. This will create a package named com.mycompany.plugins.user.

Checking the Use Minimalistic Template will cause your new plugin template to contain no extra methods or code, except 'GetVersion', and the neccessary DroidScript code. I would suggest creating a normal plugin first to learn how things work under the hood

Touch the Create Plugin button.

All files neccessary for creating a DS plugin will now be created in the AIDE working directory, /sdcard/AppProjects. All files will be updated to use your plugin name and package name.

Close this script and open AIDE

Open the file selector, find your plugin directory and select 'Open this Project'. The project will open. Select the src folder and open your plugin .java file. You will need to use the file selector again, find the 'plugin' folder inside of your project. Open the '.inc' file (Example: MyPlugin.inc). This is the JavaScript required to integrate your plugin with DS. Any functions you add to your plugin must be called from this file.

Program your plugin.

Any libraries you need AIDE to use with your plugin, copy them into the 'libs' folder inside of your project.

In the 'plugin' folder you will find a '.html' file with your plugin name. This is the file containing your plugin's documentation.

When ready to try your plugin, press the menu then, More..., Project..., Refresh Build. After this step is complete exit AIDE and return to this script.


Zipping and Installing the DS Plugin

Select Build DS Plugin Zip from the menu. A selection box will list all the project directories in your AIDE environment. Select your plugin.

Touch the Build/Install Plugin button

Your plugin '.jar' file will be created, all files within the plugin directory will be zipped into a file named after your plugin (Example: MyPlugin.zip). This script will copy it to the DS Plugins directory. You will be instructed to restart DroidScript. When you do so, your plugin will be installed.

All Done!


Reinstalling

As you work on your plugin you can use the Build DS Plugin Zip option over and over without deleting any files. It will update your plugin as needed. Remember you still must restart DroidScript between builds. Also, remember to always 'Refresh Build' from within AIDE.

I would recommend turning on Dexer Optimization within AIDE. It will cause a longer build time, but your plugin will be smaller in size and optimized.
